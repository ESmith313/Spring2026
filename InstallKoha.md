# Install Koha

## Problem
 - The biggest issue I ran into was accidentally installing my Koha OPAC on the wrong VM. Since the setup involves system-level changes, I couldn’t just move it over, so I had to completely restart the process on the correct VM. That definitely slowed me down and forced me to go back through all the steps again.
 - I also had some trouble getting Koha to work alongside my WordPress and Omeka sites. There were conflicts with ports and Apache settings that affected how everything was running. I was able to fix my WordPress site by adjusting the configuration, but I’m still working on getting Omeka functional again.

## What Surprised Me
 - What surprised me the most was how much installing Koha affected everything else on the server. I expected to just “add” a new system, but it actually changed how web traffic was handled. It made me realize how important it is to keep track of ports and server settings.

## Real World Relevance
 - This assignment made it clear that systems like Koha don’t run on their own in real life. They have to share a server with other tools, which can lead to conflicts if things aren’t set up carefully. Knowing how to troubleshoot and manage these issues is really important for keeping everything running smoothly.
